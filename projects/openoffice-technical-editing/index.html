    Apache OpenOffice Technical Editing

    This sample highlights my technical editing work on Apache OpenOffice user documentation. The project demonstrates my ability to review and improve large documentation sets for clarity, consistency, usability, structure, and technical accuracy.

    Overview

    My editing work focused on major OpenOffice user guides, including:

    • Calc 4.1 User Guide
    • Draw 4.1 User Guide
    • Impress 4.1 User Guide

    These projects required careful editorial review of end-user documentation to improve terminology, readability, organization, and overall documentation quality.

    My Role

    I served as a technical editor for Apache OpenOffice user documentation. My work included reviewing and editing content for clarity, consistency, readability, usability, and technical accuracy across multiple user guides.

    Editing Focus Areas

    This sample demonstrates several forms of editing, including:

    • Consistency editing — standardizing terminology, usage, and formatting
    • Copyediting — improving grammar, punctuation, sentence structure, and readability
    • Content editing — clarifying meaning, improving completeness, and refining wording
    • Structural editing — improving organization, flow, headings, and presentation
